Fiber Churn Intelligence

This project is a simple telecom churn prediction POC. It generates realistic customer data, trains a machine learning model, exposes an API for predictions, and supports business-friendly threshold tuning for retention campaigns.

Data generation Model training FastAPI prediction Campaign threshold tuning

1. What the project does

  • Creates a telecom customer dataset with churn-related signals.
  • Builds features like outages, complaints, billing, usage, and contract behavior.
  • Trains a churn model using scikit-learn.
  • Serves predictions with FastAPI at /predict.
  • Supports threshold tuning for retention campaigns using /threshold-tuning.

2. Main files

  • generate_dataset.py — creates the customer CSV.
  • train_model.py — trains and evaluates the model.
  • app/main.py — FastAPI service for predictions.
  • threshold_tuning.json — business threshold recommendations.

3. Flow of the system

  1. Generate customer records.
  2. Engineer useful features such as service stress, billing risk, and usage drop.
  3. Train a model and compare candidate algorithms.
  4. Save the best model and threshold settings.
  5. Use the API to estimate churn risk for any customer.

{
  "plan": "Premium",
  "monthlyPrice": 1499,
  "tenureMonths": 12,
  "outages": 2,
  "complaints": 1,

  "latePayments": 0,

  "speedMbps": 500,
  "avgMonthlyUsageGb": 180,
  "recentPlanChange": 0,

This is the type of input the model uses to estimate churn probability.

5. Example API response

{
  "churn_prediction": 1,
  "churn_probability": 0.529,
  "threshold_used": 0.1,
  "campaign": "retention_campaign",
  "risk": "MEDIUM",
  "risk_reason": "Moderate churn likelihood."
}

A higher probability means the customer is more likely to churn. Thresholds help you tune this for campaign use cases.

6. Why threshold tuning matters

Different business goals need different cutoffs:

  • Retention campaign: catch more likely churners, even if some false alarms happen.
  • Balanced: normal compromise between precision and recall.
  • Conservative: focus on the most certain high-risk customers.

7. Quick start

python generate_dataset.py
python train_model.py
python -m uvicorn app.main:app --host 127.0.0.1 --port 8000

After starting the app, open /docs for Swagger UI.

8. Sample business use case

Suppose you want to run a retention offer for customers likely to leave in the next few months. You can choose a lower threshold to flag more candidates, then contact the highest-risk ones first.

Outcome: better targeting, improved retention, and lower wasted campaign spend.
